TestFirst · 测试优先
追觅俞浩的极小试错决策系统

"与其洞察，不如测试。" —— 俞浩，追觅科技创始人


这是什么
这是一个可以直接放进AI工具使用的Skill文件。
它把追觅科技创始人俞浩8年年均100%增长背后的决策方法，拆解成普通创业者今天就能用的操作步骤。
不是人物传记。不是商业分析。
是一套可运行的决策系统——放进Claude、ChatGPT或任何支持System Prompt的AI工具，它就会用俞浩的思维框架帮你分析问题。

为什么做这个
2026年五一假期，俞浩单日发布117条视频刷屏全网。
大多数人以为他在做营销。
但他说了一句话："我不知道哪条视频会火。我认为会火的反而不火，我认为不会火的反而爆了。"
他不是在炒作。他在做实验。
2.2万名员工同时发视频，是2.2万个并行实验同时在跑——用真实数据测试平台算法，找到规律，形成SOP，再量化复制。
这背后是追觅所有产品决策的底层逻辑：极小试错半径。
不凭感觉押注，用最小成本测试，让市场数据来修正方向。
这套方法让追觅在竞争对手疯狂降价到499美元的时候，以1999美元的定价推出割草机器人，反而卖爆。2026年Q1销量同比增长174%。

怎么用
方法一：放进 Claude Projects（推荐）

打开 Claude.ai → Projects → 新建项目
把 SKILL.md 的全部内容复制到「Project Instructions」里
在这个项目里开始对话，直接问：

帮我用N+1方法分析这个产品方向：[你的想法]
我在做[品类]，帮我找用户最核心的一个痛点
用俞浩的极小试错逻辑，帮我设计一个验证方案
方法二：放进 OpenClaw / Hermes

下载 SKILL.md
放入你的 .claude/skills/ 文件夹
触发词：俞浩视角、N+1分析、极小试错

方法三：直接复制到任何AI工具
把 SKILL.md 的内容粘贴到对话开头，作为System Prompt使用。

适合谁
✅ 早期创业者 — 资源有限，不能犯大错，需要在验证前找到最小试错路径
✅ 产品经理 — 需要在上线前验证假设，而不是靠直觉做决策
✅ 自媒体创作者 — 不知道什么内容会火，需要系统化测试而不是凭感觉发布
✅ 硬件/消费品从业者 — N+1方法论在有成熟竞品的赛道效果最好

不适合谁
❌ 纯软件/互联网产品（俞浩经验集中在硬件制造）
❌ 需要0→1颠覆式创新的场景（N+1建立在成熟N的基础上）
❌ 已经有充分市场验证数据的成熟业务
