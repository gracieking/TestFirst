## 心智模型（5个）

### 1. 极小试错半径
**核心**：用最小成本验证假设，不在验证前大规模投入

> "追觅希望用最小试错成本，对业务可能性进行快速验证、并持续迭代修正，这比直接把错误产品推向市场好。"

**操作方式**：
- 先小范围验证，再大规模复制
- 验证的标准是真实的购买行为，不是用户的口头意愿
- 失败了修正再测，不押注式投入

**真实案例**：
俞浩单日发布117条视频，不是因为他知道哪条会火——他自己说他不知道。他在用高频发布测试平台推荐算法，让数据告诉他规律，找到SOP后量化复制给2.2万名员工。这是极小试错半径在自媒体场景的直接应用。

**局限性**：
在需要快速抢占市场、先发优势极强的场景，过于保守的试错可能错过窗口期。

---

### 2. N+1创新方法论
**核心**：在成熟产品（N）基础上，用技术长板解决那一个核心痛点（+1）

> "N+1的核心逻辑是在行业成熟标杆的基础上，找准一个用户核心痛点，用追觅的技术长板打出代际差。用新技术重做成熟品类，成功率远高于在主流赛道追求颠覆式发明。"

**操作三步**：
1. 找到行业里用户最频繁抱怨的那一个问题（不是前三个，是那一个）
2. 判断你有没有技术或能力比现有玩家更好地解决它
3. 用高于市场的定价验证——如果真的解决了核心问题，价格不是障碍

**真实案例**：
割草机行业最大痛点：边界识别不准。主流定价499-999美元，行业在卷价格。追觅预判车用激光雷达将快速降价，率先引入，攻克边界识别问题，以1999美元（行业主流2倍）上市，反而卖爆。2026年Q1销量同比增长174%，波兰市占率登顶第一。

**技术复用链**：
高速数字马达 → 扫地机 → 割草机 → 汽车电驱 → 机器人 → 无人机
机械臂技术 → 扫地机边角清洁 → 空调送风 → 洗衣机旋钮 → 厨电搅拌

**局限性**：
只适用于已有成熟模板的领域。真正的0→1原始创新，N+1不适用。

---

### 3. 物理学思维（第一性原理）
**核心**：不追风口，不赌概率，回到基础规律推演

> "物理学思维的本质是尊重第一性原理，不追风口，不赌概率。物理学不是做完所有实验，而是通过两个点抽象出公式，适用所有领域。"

**操作方式**：
- 不看竞争对手做了什么，而是回到问题的物理原理推导
- 用两个数据点推导出趋势曲线，而不是做完所有调研再判断
- 决策依据是规律，不是感觉

**真实案例**：
追觅判断车用激光雷达会快速降价（两个数据点：汽车行业规模化 + 零部件降价规律），提前布局割草机应用，而不是等市场验证后再跟进。

**局限性**：
当行业发生根本性范式转变（新技术路线颠覆原有物理基础）时可能失效。需警惕过度依赖历史规律。

---

### 4. 左右模型
**核心**：经营能力（左）和社会资源（右）必须两边都强

> "追觅有一个左侧模型和右侧模型。左侧关于如何做好经营；右侧是如何获得社会资源。贾跃亭的问题是，他几乎完全活在右侧，他没挣过钱，乐视从头到尾都在亏损，而追觅的主业一直是盈利的。"

**判断标准**：
- 左侧（经营）：主营业务是否持续盈利？现金流是否健康？
- 右侧（资源）：品牌、融资、政府关系、生态合作伙伴
- 只靠右侧会死（乐视）；只靠左侧会慢（错过窗口期）

**局限性**：
不同阶段的左右比重不同，早期创业者过度强调左侧可能导致增长过慢。

---

### 5. 工程师修正系统
**核心**：把直觉变成可验证的假设，建立纠错机制

> "很多CEO所谓的直觉，跟一个优秀工程师的判断，能力差距没那么大。区别在于——CEO的错误没人修正，工程师的错误会被系统修正。"

**操作方式**：
- 不依赖直觉做大决策，而是把直觉转化成假设
- 用极小试错半径验证假设
- 让市场数据而不是个人判断来修正方向

## 决策检查表（6问）

在做任何重要决策前，先问自己这6个问题：

```
□ 1. 我是在凭洞察判断，还是有真实数据支撑？
      → 如果是洞察，先用最小成本测试再决定

□ 2. 我解决的是用户最核心的那一个问题，还是在做大而全？
      → 缩到一个痛点，找到N+1的切入点

□ 3. 这个+1，我有没有技术或能力上的差异化？
      → 如果没有，你在做的是跟随，不是创新

□ 4. 我的验证方式是真实购买行为，还是用户口头反馈？
      → 用户说"我会买"≠他会真的买，只有付款才算验证

□ 5. 我的主营业务（左侧）能支撑这个新方向的试错成本吗？
      → 在左侧没稳住之前，谨慎扩张右侧

□ 6. 如果这个决策是错的，我有没有足够小的代价来修正？
      → 如果犯错代价太大，说明试错半径不够小
```

## N+1三步SOP（可直接使用）

**第一步：找那一个痛点**
```
去哪里找：
- 京东/亚马逊差评区（按"最差"排序，读100条）
- 小红书搜索"[品类名] 踩坑 / 后悔 / 问题"
- 竞品的一星评价

找什么：
- 用户最频繁、最情绪化提到的那一个问题
- 不是前三名，是出现频率最高的那一个
- 记录原话，不要翻译成抽象概念
```

**第二步：判断你的+1**
```
问自己：
- 有没有一个技术趋势，让这个问题在近期可以被更好地解决？
- 我的专业背景/供应链/资源，有没有让我比别人更早、更好解决这个问题的能力？
- 如果真正解决了这个问题，用户愿意为此多付多少钱？

判断标准：
- 你的+1必须是别人在短期内难以复制的
- 不是"我做得更好"，是"我解决了他们根本没解决的问题"
```

**第三步：极小试验**
```
验证方式（从低成本到高成本依次尝试）：
1. 发一条视频/帖子，看有没有人问"在哪里买/多少钱"
2. 发预售链接，看有没有人愿意付定金
3. 小批量生产，用真实定价在小范围上市
4. 根据反馈修正，再测

验证标准：
- 通过：真实购买行为出现，且复购意愿强
- 失败：有人感兴趣但不付钱（说明价值没到位）
- 修正：找到付钱/不付钱的分界点，调整产品或定价

不要做的：
- 在验证通过之前大规模生产
- 用"用户说会买"代替真实购买数据
- 因为"感觉对了"就加速投入
```

## 自媒体增长SOP（俞浩刷屏逻辑拆解）

俞浩单日117条视频的本质不是营销，是实验：

```
第一步：高频发布（建立数据样本）
- 每天发，不筛选，不等"准备好了"
- 目标：积累足够多的样本，让算法有数据可以推断

第二步：追踪数据（找规律）
- 记录每条视频的完播率、互动率、涨粉数
- 不看绝对数字，看相对规律：什么主题/形式/时间发布的数据更好

第三步：提炼SOP（固化规律）
- 把表现好的内容拆解：标题结构、开场方式、内容类型、时长
- 写成可复制的操作规范

第四步：量化复制（规模化）
- 把SOP教给更多人执行（俞浩是2.2万员工）
- 你是一个人，可以是：AI辅助生产、固定模板、批量录制

核心原则：
你不需要知道哪条会火。你需要的是足够多的试验，让数据告诉你规律。
```

## 诚实边界

**这个Skill无法做到的事：**

- 无法预测全新赛道：俞浩经验集中在硬件/消费电子，纯软件/互联网领域参考价值有限
- 无法替代你对行业的深度认知：N+1需要你真正懂那个N是什么
- 无法保证成功：极小试错降低失败成本，不消除失败概率
- 信息截止于2026年5月，之后的动态未知
